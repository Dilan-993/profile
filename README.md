# Первое знакомство с Git

Git — это система контроля версий.

Она помогает сохранять историю проекта.

Можно посмотреть:

* какие изменения были сделаны;
* когда они были сделаны;
* кем они были сделаны.

Git можно сравнить с сохранениями в компьютерной игре.

После каждого важного этапа мы будем делать "сохранение" проекта.

Такое сохранение называется **коммитом**.

---

# Команды Git

## git init

```bash
git init
```

Создает Git-репозиторий внутри проекта.

После выполнения этой команды Git начинает отслеживать изменения файлов.

Эта команда выполняется только один раз — при создании проекта.

---

## git remote add origin

```bash
git remote add origin <ссылка_на_репозиторий>
```

Подключает локальный проект к удаленному репозиторию GitHub.

Проще говоря:

* проект находится на компьютере;
* GitHub находится в интернете;
* эта команда "знакомит" их между собой.

---

## git add

```bash
git add .
```

Добавляет все измененные файлы в список файлов для следующего коммита.

Можно представить это как корзину перед сохранением.

Мы говорим Git:

> "Вот эти изменения нужно сохранить."

---

## git commit

```bash
git commit -m "Организовали структуру проекта"
```

Создает новую точку сохранения проекта.

Текст после `-m` называется **сообщением коммита**.

По нему можно понять, что именно изменилось.

---

## git push

```bash
git push -u origin master
```

Отправляет все коммиты на GitHub.

После этого изменения появляются в удаленном репозитории.

> **Примечание.**
> Во многих современных проектах вместо ветки `master` используется ветка `main`. Если репозиторий был создан с веткой `main`, то команда будет выглядеть так:
>
> ```bash
> git push -u origin main
> ```

# Что такое HTML?

**HTML (HyperText Markup Language)** — это язык разметки.

Он нужен для создания структуры веб-страницы.

Если сравнить сайт с домом, то HTML — это его каркас.

Именно HTML сообщает браузеру:

* где находится заголовок;
* где будет текст;
* где расположено меню;
* где появится изображение;
* где будут ссылки и кнопки.

Сам по себе HTML не делает страницу красивой.

За внешний вид отвечает CSS, а за поведение сайта — JavaScript.

---

# Структура HTML-документа

Каждая HTML-страница начинается с объявления типа документа.

```html
<!DOCTYPE html>
```

Эта строка сообщает браузеру, что перед ним современный HTML-документ.

---

После этого начинается сам документ.

```html
<html lang="ru">
```

Тег `<html>` является главным тегом всей страницы.

Внутри него находится абсолютно всё содержимое сайта.

Атрибут `lang="ru"` сообщает браузеру, что страница написана на русском языке.

Это помогает:

* браузеру;
* поисковым системам;
* программам для чтения текста.

---

# Тег `<head>`

Внутри `<head>` хранится служебная информация.

Она почти никогда не отображается на самой странице.

В нашем проекте находятся:

```html
<head>

    <meta charset="UTF-8">

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <title>Вася Пупкин * начинающий программист</title>

</head>
```

Разберем каждый элемент.

---

## `<meta charset="UTF-8">`

Указывает кодировку страницы.

Благодаря этому русский текст отображается правильно.

Если забыть эту строку, вместо букв могут появиться непонятные символы.

---

## `<meta name="viewport"...>`

Этот тег нужен для правильного отображения сайта на телефонах и планшетах.

Без него сайт может выглядеть слишком маленьким или неудобным на мобильных устройствах.

Практически все современные сайты используют этот тег.

---

## `<title>`

Определяет название страницы.

Этот текст отображается:

* на вкладке браузера;
* в закладках;
* в результатах поисковых систем.

В нашем проекте:

```html
<title>Гриша Галатов * начинающий программист</title>
```

---

# Тег `<body>`

Внутри `<body>` располагается всё, что видит пользователь.

Именно здесь создается содержимое сайта.

Все тексты, изображения, кнопки и меню находятся внутри этого тега.

# Тег `<section>`

Во время урока мы несколько раз использовали тег

```html

```

`<section>` переводится как **раздел**.

Этот тег помогает разделить страницу на логические части.

Можно представить страницу сайта как книгу.

Тогда:

* каждая глава книги — это `<section>`;

Такой подход делает код более понятным и удобным для чтения.

# Атрибут `id`

Во время урока появился новый атрибут:

```html
id="about"
```

Раньше мы уже использовали атрибут `class`.

Теперь познакомились еще с одним важным атрибутом — `id`.

Главное отличие:

* `class` можно использовать много раз;
* `id` должен быть уникальным.

На одной странице не должно быть двух одинаковых `id`.

---

# Зачем нужен `id`?

В начале страницы мы уже создали меню.

Например:

```html
<a href="#about">Обо мне</a>
```

Здесь значение `href="#about"` совпадает со значением

```html
id="about"
```

Когда пользователь нажимает на ссылку, браузер автоматически находит элемент с таким `id` и переходит к нему.

Именно поэтому меню работает без использования JavaScript.

# Почему используются одинаковые `class`?

Мы создали несколько элементов

```html
class="card"
```

Это сделано специально.

Все карточки должны выглядеть одинаково.

Когда мы начнем изучать CSS, достаточно будет написать стиль только один раз.

Он автоматически применится ко всем карточкам.

Это одна из главных особенностей использования `class`.

* создавать ссылки на электронную почту;
* использовать тег `<footer>`;
* понимать общую структуру современного сайта.

Теперь наш сайт имеет практически готовую HTML-структуру. На следующих уроках мы начнем изучать CSS и научимся превращать обычный текст в красивую современную веб-страницу.


# Урок №4. Первое знакомство с CSS

## Цель урока

На этом занятии мы познакомились с языком **CSS** и впервые начали изменять внешний вид нашего сайта.

До этого момента HTML отвечал только за структуру страницы.

Теперь мы узнали, как сделать страницу красивой с помощью CSS.

---

# Что такое CSS?

**CSS (Cascading Style Sheets)** — это язык стилей.

Если HTML — это каркас дома, то CSS — это его отделка.

Именно CSS отвечает за:

* цвета;
* размеры текста;
* расположение элементов;
* шрифты;
* отступы;
* анимации.

Без CSS практически все сайты выглядели бы как обычный черный текст на белом фоне.

---

# Как подключить CSS?

Мы создали файл

```text
assets/css/style.css
```

После этого подключили его в файле `index.html`.

```html
<link rel="stylesheet" href="assets/css/style.css">
```

Теперь браузер знает, где находятся стили нашего сайта.

Каждый раз при открытии страницы браузер сначала загружает HTML, а затем применяет к нему правила из CSS.

---

# Как устроено правило CSS?

Практически каждое правило CSS выглядит одинаково.

```css
селектор {

    свойство: значение;

}
```

Например:

```css
body {

    background: #0d1117;

}
```

Здесь:

* `body` — селектор;
* `background` — свойство;
* `#0d1117` — значение.

Можно сказать так:

> "Найди элемент `<body>` и сделай его фон темным."

---

# Первый селектор — `*`

Первым правилом нашего файла стало:

```css
* {

    margin: 0;
    padding: 0;
    box-sizing: border-box;

}
```

Знак

```css
*
```

называется **универсальным селектором**.

Он выбирает **все элементы страницы**.

Это значит, что правило применяется сразу ко всему сайту.

---

# Свойство `margin`

```css
margin: 0;
```

`margin` — это внешний отступ элемента.

По умолчанию браузер сам добавляет небольшие отступы.

Из-за этого разные браузеры могут отображать страницу немного по-разному.

Мы убрали эти отступы, чтобы полностью управлять расположением элементов самостоятельно.

---

# Свойство `padding`

```css
padding: 0;
```

`padding` — это внутренний отступ.

Он определяет расстояние между содержимым элемента и его границами.

Пока мы также обнулили его.

Позже мы научимся использовать `padding`, чтобы красиво располагать текст внутри блоков.

---

# Свойство `box-sizing`

```css
box-sizing: border-box;
```

Это одно из самых полезных свойств CSS.

Оно изменяет способ вычисления размеров элементов.

Благодаря ему становится намного проще рассчитывать ширину и высоту блоков.

Практически во всех современных проектах используется именно

```css
box-sizing: border-box;
```

---

# Стили для `<body>`

Следующее правило:

```css
body {

    background: #0d1117;

    color: #ffffff;

}
```

Теперь мы изменили внешний вид всей страницы.

---

# Свойство `background`

```css
background: #0d1117;
```

`background` отвечает за фон элемента.

Мы сделали фон страницы темным.

---

# Что такое HEX-цвет?

В CSS существует несколько способов записи цветов.

Мы использовали запись

```css
#0d1117
```

Такой способ называется **HEX-кодом**.

Каждый цвет имеет собственный код.

Например:

```text
Белый      → #ffffff

Черный     → #000000

Красный    → #ff0000

Зеленый    → #00ff00

Синий      → #0000ff
```

Позже мы познакомимся и с другими способами задания цветов.

---

# Свойство `color`

```css
color: #ffffff;
```

Свойство `color` изменяет цвет текста.

После этого правила весь текст страницы стал белым.

Теперь белые буквы хорошо читаются на темном фоне.

---

# Что получилось?

После сегодняшнего урока сайт изменился совсем немного.

Но произошло очень важное событие:

Мы впервые отделили **структуру** страницы от ее **оформления**.

Теперь:

* HTML отвечает за содержимое сайта.
* CSS отвечает за внешний вид сайта.

Именно так создаются практически все современные веб-приложения.

---

# Итоги урока

После четвертого занятия мы научились:

* понимать назначение CSS;
* подключать файл стилей к HTML;
* понимать устройство CSS-правила;
* использовать универсальный селектор `*`;
* работать со свойствами `margin`, `padding` и `box-sizing`;
* изменять фон страницы;
* изменять цвет текста;
* познакомились с HEX-кодами цветов.

Теперь мы сделали первый шаг к созданию красивого современного сайта.


# Урок №5. Стилизация шапки сайта и меню навигации

## Цель урока

На этом занятии мы продолжили изучать CSS.

Мы научились изменять внешний вид шапки сайта, познакомились с технологией **Flexbox**, оформили меню навигации и впервые сделали элементы страницы интерактивными.

После этого урока шапка сайта стала выглядеть как на современных веб-сайтах.

---

# Повторение

На прошлом уроке мы:

* подключили файл `style.css`;
* познакомились с CSS;
* изменили фон страницы;
* изменили цвет текста.

Сегодня мы будем работать с элементами, которые создали в файле `index.html`:

```html
<header>

```

# Стили для `<header>`

Мы написали первое правило для шапки сайта.

```css
header{

    display:flex;

    padding:25px 10%;
```

Разберем каждое свойство отдельно.

---

# Свойство `display`

```css
display:flex;
```

До этого момента браузер располагал элементы друг под другом.

Свойство

```css
display:flex;
```

включает специальный режим расположения элементов — **Flexbox**.

Flexbox помогает удобно располагать элементы по горизонтали или вертикали.

После включения Flexbox логотип и меню оказались на одной линии.

---

# Что такое Flexbox?

Представьте коробку.

Внутри нее лежат несколько предметов.

Flexbox позволяет легко управлять расположением этих предметов.

Например:

* поставить рядом;
* раздвинуть;
* выровнять по центру;
* расположить справа или слева.

Сегодня мы познакомились с Flexbox впервые.

В следующих уроках мы будем использовать его очень часто.

---

# Свойство `justify-content`

```css
justify-content:space-between;
```

Это свойство управляет расположением элементов по горизонтали.

Значение

```css
space-between
```

раздвигает элементы максимально далеко друг от друга.

В нашем случае получилось так:

```text
Логотип                               Меню
```

Логотип оказался слева, а меню — справа.

---

# Свойство `padding`

```css
padding:25px 10%;
```

Мы уже знаем, что `padding` отвечает за внутренние отступы.

Здесь указаны сразу два значения.

Первое число

```text
25px
```

означает верхний и нижний отступ.

Второе значение

```text
10%
```

означает отступы слева и справа.

Благодаря этому содержимое не "прилипает" к краям страницы.

---

# Свойство `position`

```css
position:fixed;
```

Это свойство изменяет способ расположения элемента.

Значение

```text
fixed
```

закрепляет элемент на экране.

Теперь шапка сайта остается сверху даже тогда, когда пользователь прокручивает страницу вниз.

Такое решение очень часто используется на современных сайтах.

---

# Свойство `width`

```css
width:100%;
```

Ширина шапки стала равна всей ширине окна браузера.

Теперь `<header>` занимает всю верхнюю часть страницы.

---

# Свойство `background`

```css
background:#161b22;
```

Мы изменили цвет фона шапки.

Теперь она отличается от основного фона страницы.

Так меню стало более заметным.

---

# Стили для меню

Следующее правило относится к тегу `<nav>`.

```css
nav{

    display:flex;

    gap:30px;

}
```

---

# Свойство `gap`

```css
gap:30px;
```

`gap` отвечает за расстояние между элементами.

Теперь между ссылками появилось одинаковое пространство.

Без этого свойства все ссылки располагались бы вплотную друг к другу.

---

# Стили для ссылок

Мы изменили внешний вид ссылок меню.

```css
nav a{

    text-decoration:none;

    color:white;

    transition:.3s;

}
```

---

# Свойство `text-decoration`

```css
text-decoration:none;
```

По умолчанию браузер подчеркивает ссылки.

Это свойство убирает подчеркивание.

Теперь ссылки выглядят аккуратнее.

---

# Свойство `color`

```css
color:white;
```

Изменяет цвет текста.

Мы сделали ссылки белыми.

---

# Свойство `transition`

```css
transition:.3s;
```

Появилось новое свойство.

Оно отвечает за плавность изменения.

Если изменить цвет без `transition`, это произойдет мгновенно.

С `transition` изменение занимает 0.3 секунды и выглядит намного красивее.

---

# Псевдокласс `:hover`

Последнее правило сегодняшнего урока:

```css
nav a:hover{

    color:#58a6ff;

}
```

Псевдокласс

```text
:hover
```

срабатывает тогда, когда пользователь наводит курсор мыши на элемент.

В нашем случае ссылка меняет цвет.

Так пользователь понимает, что именно на эту ссылку сейчас наведен курсор.

Практически все современные сайты используют подобный эффект.

---

После сегодняшнего урока сайт изменился очень заметно.

* ссылки аккуратно разделены;
* шапка закреплена в верхней части страницы;
* ссылки реагируют на наведение мыши.

Сайт стал выглядеть гораздо современнее.
