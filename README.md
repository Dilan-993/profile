На первом занятии мы не писали код сайта. Наша задача была научиться правильно организовывать проект, чтобы в дальнейшем в нем было легко ориентироваться.

После первого урока проект имеет следующий вид:

```text

│

├── README.md
├── index.html
├── req.txt
│

    ├── css/
    ├── js/
    └── img/
```

Теперь разберем назначение каждого файла и каждой папки.

## assets

Название **assets** переводится как:

* ресурсы;
* материалы;
* дополнительные файлы проекта.

Эта папка нужна для хранения файлов, которые используются сайтом.

Мы сразу создали такую структуру, чтобы привыкать к правильной организации проекта.

Именно CSS отвечает за внешний вид сайта:

* размеры;

## .gitignore

Git отслеживает изменения файлов.

Но некоторые файлы хранить в репозитории не нужно.

Для этого существует файл `.gitignore`.

В нем перечисляются файлы и папки, которые Git должен игнорировать.

Сейчас в нем находится:

```text

.venv
__pycache__/

```

# Первое знакомство с Git

Git — это система контроля версий.

Она помогает сохранять историю проекта.

Можно посмотреть:

* какие изменения были сделаны;
* когда они были сделаны;
* кем они были сделаны.

Git можно сравнить с сохранениями в компьютерной игре.

После каждого важного этапа мы будем делать "сохранение" проекта.

Такое сохранение называется **коммитом**.

---

# Команды Git

## git init

```bash
git init
```

Создает Git-репозиторий внутри проекта.

После выполнения этой команды Git начинает отслеживать изменения файлов.

Эта команда выполняется только один раз — при создании проекта.

---

## git remote add origin

```bash
git remote add origin <ссылка_на_репозиторий>
```

Подключает локальный проект к удаленному репозиторию GitHub.

Проще говоря:

* проект находится на компьютере;
* GitHub находится в интернете;
* эта команда "знакомит" их между собой.

---

## git add

```bash
git add .
```

Добавляет все измененные файлы в список файлов для следующего коммита.

Можно представить это как корзину перед сохранением.

Мы говорим Git:

> "Вот эти изменения нужно сохранить."

---

## git commit

```bash
git commit -m "Организовали структуру проекта"
```

Создает новую точку сохранения проекта.

Текст после `-m` называется **сообщением коммита**.

По нему можно понять, что именно изменилось.

---

## git push

```bash
git push -u origin master
```

Отправляет все коммиты на GitHub.

После этого изменения появляются в удаленном репозитории.

> **Примечание.**
> Во многих современных проектах вместо ветки `master` используется ветка `main`. Если репозиторий был создан с веткой `main`, то команда будет выглядеть так:
>
> ```bash
> git push -u origin main
> ```
